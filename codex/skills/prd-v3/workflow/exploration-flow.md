# 探索模式 3 步流程

> 探索备忘录的轻量流程，不走前置评审/流程规划/蓝图/评分。

## Step E1: 需求理解（简版）

AI分析用户输入，提取核心问题：
- 不做推荐引擎计算（关键词直配直接选择模式）
- 不走 prd-requirement-review 前置步骤
- 直接开始填写模板

**输入**：用户的自然语言描述（"我想探索XX"/"要不要做XX"/"值不值得做XX"）

**行为**：
1. 提取核心问题（痛点、假设、愿景方向）
2. 判断是否需要AI项目专用模板（检测关键词：智能体/Agent/AI/对话/聊天）
3. 选择模板：standard.md 或 ai-agent.md

## Step E2: 生成探索备忘录

填充模板 §1-§3：
- §1 从用户输入提取痛点、假设、愿景
- §2 AI自动补充可行性评估维度（技术/经济/组织）
- §3 AI给出推荐路径建议

**保存路径**：`output/[项目名]/exploration/[项目名]-exploration.md`

## Step E3: 简版确认

展示生成内容，用户确认或调整：
- 不做评分（探索备忘录不做量化评分）
- 不做蓝图/drawio/HTML（探索阶段不需要视觉产出）
- 如果确认"值得做" → 提示用户可以进入PRD流程
- 如果确认"暂缓" → 保存备忘录，流程结束
- 如果需要调整 → 用户修改后重新保存
