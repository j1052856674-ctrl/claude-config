# 智能推荐行为规则

> 定义 Claude 在推荐模式/模板/图表组合时的行为规则。
> 核心原则：**推荐而非反复提问**，让用户快速起步而非陷入选择困境。
> 引用配置：`config/engine.json`、`knowledge/diagrams/index.json`、`knowledge/templates/shared-index.json`

---

## 核心行为规则

1. **推荐而非反复提问**：当用户描述足以推断 mode 和 industry 时，直接推荐最匹配选项 + 理由，而非列举所有选项让用户选择

2. **推荐话术模板**：推荐结果必须包含理由说明，格式如下：

   ```
   推荐使用 {mode}模式 + {industry}行业模板，原因：
   1) {行业关键词匹配理由}
   2) {复杂度匹配理由}
   3) {用户偏好匹配理由，如有}
   如果不合适，可以选择其他组合：{列出备选}
   ```

3. **图表组合推荐**：基于项目描述推断复杂度，自动推荐图表类型组合（引用 `knowledge/diagrams/index.json` 推荐矩阵）

4. **子模板触发推荐**：基于关键词自动判断是否需要引用特定子模板（引用 `knowledge/templates/shared-index.json` 触发关键词）

---

## 推荐引擎计算流程

> 引用配置：`config/engine.json`

```
1. 提取用户输入关键词
2. 匹配行业关键词 → industry_match 评分
3. 匹配模式关键词 → complexity_match 评分
4. 查询用户画像偏好 → user_preference 评分（首次使用则此项为0）
5. 查询模板热度 → popularity 评分（usage_count）
6. 加权计算总分 = 0.4×industry + 0.3×complexity + 0.2×preference + 0.1×popularity
7. 总分 ≥ 0.7 的模板按分数排序推荐
```

**计算参数说明：**

| 因子 | 权重 | 来源 | 说明 |
|------|------|------|------|
| 行业匹配 | 0.4 | config/engine.json industry_keywords | 匹配风控/电商/金融/数据智能关键词 |
| 复杂度匹配 | 0.3 | config/engine.json mode_keywords | 匹配完整/轻量关键词 |
| 用户偏好 | 0.2 | knowledge/profile.json | 查询用户历史选择偏好 |
| 热度 | 0.1 | knowledge/templates/index.json | 查询模板 usage_count |

**推荐阈值：**

- 相似度阈值 ≥ 0.7 才推荐
- 最大推荐数 = 3

## 图表推荐矩阵

> 引用配置：`knowledge/diagrams/index.json`

| 产品复杂度 | 推荐图表组合 | 最少数量 | 最多数量 |
|-----------|-------------|---------|---------|
| 简单 | 3层架构图 + 流程图 + 时序图 | 2 | 3 |
| 中等 | 5层架构图 + 业务流程图 + 时序图 | 2 | 4 |
| 复杂 | 5-7层架构图 + 全套（架构+业务流+时序+数据+部署） | 3 | 5 |

## 子模板触发关键词

> 引用配置：`knowledge/templates/shared-index.json`

| 子模板 | 触发关键词 | 适用行业 |
|--------|-----------|---------|
| Agent对话规格 | Agent、对话、智能客服、bot、聊天机器人、脚本 | 风控/通用 |
| 数据字段定义规格 | 字段、数据、特征、宽表、指标、计算逻辑 | 风控/通用 |
| 交互规格 | 交互、按钮、输入、列表、上传、页面、UI | 风控/通用 |
| 风险分析规格 | 风险、异常、容错、降级、回滚、安全 | 风控/通用 |
| 数据埋点规格 | 埋点、追踪、统计、曝光、点击、分析 | 风控/通用 |

当用户需求描述中出现以上关键词时，自动在推荐话术中提示对应的子模板引用。
