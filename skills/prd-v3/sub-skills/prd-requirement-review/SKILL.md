# 需求评审 Skill

## 定位

PRD 流程的前置步骤。输入用户需求描述，输出需求评审报告 + 关键决策点。

**核心价值**：帮助用户把需求想清楚，避免"拍脑袋"做产品。

## 执行流程（7步）

### Step 1: 复述理解

复述用户对需求的理解，确认是否正确。

```
我理解您的需求是：
- 产品类型: [智能推断]
- 核心诉求: [从需求描述提取]
- 目标用户: [推断]
- 使用场景: [推断]

请确认或补充。
```

### Step 2: 追问澄清（5W1H）

| 维度 | 问题 | 作用 |
|------|------|------|
| **What** | 核心功能是什么？ | 明确产品是什么 |
| **Who** | 目标用户是谁？ | 明确给谁用 |
| **When** | 什么时候用？ | 明确使用场景 |
| **Where** | 在哪里用？ | 明确使用环境 |
| **Why** | 为什么要做这个？ | 明确业务价值 |
| **How** | 用户怎么用？ | 明确交互方式 |

### Step 3: 目标用户识别

输出目标用户画像：

```markdown
## 目标用户

| 维度 | 描述 |
|------|------|
| 角色 | 策略人员 / 人审人员 / 产品经理 |
| 痛点 | 他们当前面临的痛点是什么？ |
| 需求 | 他们需要什么？ |
| 使用频率 | 高频 / 低频 |
```

### Step 4: 价值评估

```markdown
## 价值评估

| 维度 | 评估 | 说明 |
|------|------|------|
| 业务价值 | 高/中/低 | 解决了什么业务痛点？能带来的收益？ |
| 用户价值 | 高/中/低 | 目标用户有多大？用户需求的迫切程度？ |
| 技术可行性 | 高/中/低 | 技术难度如何？是否可实现？ |
| 竞争差异化 | 有/无 | 与竞品相比有什么优势？ |
| 投入产出比 | 高/中/低 | 投入成本 vs 预期收益 |
```

### Step 5: 合理性检查

检查需求是否合理：

```markdown
## 合理性检查

| 检查项 | 结果 | 说明 |
|--------|------|------|
| 需求是否明确 | ✅/❌ | |
| 目标用户是否清晰 | ✅/❌ | |
| 使用场景是否完整 | ✅/❌ | |
| 技术是否可行 | ✅/❌ | |
| 是否有竞品参考 | ✅/❌ | |
```

### Step 6: 竞品参考（可选）

快速搜索1-2个类似产品，提供参考。

```markdown
## 竞品参考

| 产品 | 功能 | 优点 | 缺点 |
|------|------|------|------|
| 竞品A | ... | ... | ... |
| 竞品B | ... | ... | ... |
```

### Step 7: 关键决策点输出

输出需要用户确认的关键决策点：

```markdown
## 关键决策点（请确认）

| # | 决策点 | 默认方案 | 备选方案 | 影响 |
|---|--------|---------|---------|------|
| 1 | 角色选择 | 前置选择 | 不选择 | 影响查询逻辑 |
| 2 | 查询方式 | 一键查询 | 输入查询条件 | 影响用户体验 |
| 3 | 意图识别 | 追问阶段 | 主流程 | 影响流程设计 |
| 4 | 追问机制 | 页面内追问 | 跳转新页面 | 影响交互设计 |

请确认以上决策点，确认后进入流程规划。
```

## 输出

```
output/[项目名]/requirement-review/
├── [项目名]-requirement-review.md    # 需求评审报告
└── [项目名]-decisions.md              # 关键决策点
```

## 与 prd-flow-planning 的关系

```
用户描述需求
    ↓
调用 prd-requirement-review
    ├─ 复述理解
    ├─ 追问澄清（5W1H）
    ├─ 目标用户识别
    ├─ 价值评估
    ├─ 合理性检查
    ├─ 竞品参考（可选）
    └─ 输出关键决策点
    ↓
用户确认关键决策点
    ↓
调用 prd-flow-planning
    ↓
产出流程规划文档
```

## 与 prd-v3 的关系

```
用户描述需求
    ↓
prd-requirement-review（需求评审）
    ↓
prd-flow-planning（流程规划）
    ↓
prd-v3（PRD生成）
    ↓
drawio-skill（图纸生成）
```
