# 风控安全合规标准

> 本文件定义了风控项目PRD中安全与合规章节的标准内容。
> 所有风控项目的PRD都必须包含这些安全要求，可根据项目特点增减。

---

## 一、数据脱敏标准

### 必须脱敏的字段类型

| 字段类型 | 脱敏方式 | 示例 |
|---|---|---|
| 身份证号 | 仅显示前3后4，中间用*替代 | 430***1234 |
| 手机号 | 仅显示前3后4，中间用*替代 | 138***5678 |
| 姓名 | 仅显示姓氏，名用*替代 | 张* |
| 银行卡号 | 仅显示后4位，前面用*替代 | ***4567 |
| 地址 | 仅显示省市，详细地址用*替代 | 广东省深圳市*** |
| 银行卡CVV | 完全脱敏，不显示任何信息 | *** |

### 脱敏规则

1. **入模型前脱敏**：敏感字段在进入大模型前必须完成脱敏，脱敏标准参照公司数据安全规范
2. **入系统前脱敏**：敏感字段在写入非安全系统前必须脱敏
3. **展示脱敏**：前端展示敏感信息时必须脱敏
4. **日志脱敏**：操作日志中记录的敏感信息必须脱敏
5. **不可逆脱敏**：脱敏后不可通过反推还原原始数据

## 二、权限隔离标准

### 用户权限原则

| 原则 | 说明 |
|---|---|
| 数据隔离 | 用户之间对话任务隔离，用户仅能查看自己的对话任务信息 |
| 操作隔离 | 不同角色的用户只能执行其权限范围内的操作 |
| 角色矩阵 | 明确各角色的可看/可操作范围 |

### 常见角色权限矩阵

| 角色 | 可看 | 可操作 |
|---|---|---|
| 风控策略人员 | 自己的对话任务、规则配置页面 | 新建对话、配置规则、上线规则 |
| 风控研发 | 所有系统配置、技术日志 | 系统配置、接口调试 |
| 风控管理层 | 所有团队对话任务统计、规则效果汇总 | 审批规则上线 |
| 运营人员 | 埋点数据、用户反馈统计 | 查看统计、导出数据 |

### 权限校验机制

1. **页面级**：未授权用户访问页面时跳转/提示
2. **操作级**：未授权操作时禁用按钮+提示
3. **数据级**：查询参数校验数据归属，防止横向越权

## 三、操作记录与审计标准

### 必须记录的操作类型

| 操作类型 | 记录内容 | 保留时长 |
|---|---|---|
| 对话操作 | 用户输入内容、Agent回复内容、时间、任务ID | 6个月 |
| 规则配置 | 配置人、配置内容、审批人、审批结果 | 永久 |
| 规则上线/下线 | 操作人、规则ID、上线时间、下线时间 | 永久 |
| 数据访问 | 访问人、访问时间、访问数据范围 | 6个月 |
| 审批操作 | 审批人、审批结果、审批时间、审批意见 | 永久 |

### 审批节点要求

> 参照《乐信授权管理制度》

1. **所有对业务生产数据产生影响的变更操作**，必须设计审批节点
2. 审批节点与审批系统交互
3. 审批结果记录可追溯

## 四、安全介入评估

### 需要安全人员介入的判断标准

| 判断维度 | 是否需要安全介入 |
|---|---|
| 涉及用户敏感数据（身份证/手机号/银行卡） | **是** |
| 涉及大模型处理用户数据 | **是** |
| 涉及规则对用户资金产生直接影响 | **是** |
| 涉及新接口对外开放 | **是** |
| 仅内部配置变更，不涉及用户数据 | 视情况 |

### 安全介入流程

1. 需求提出人设置"是否须安全介入"为"是"
2. 系统通过企微通知信息安全人员
3. 安全人员评估后补充安全详情
4. 补充的安全需求作为需求的一部分进行评审、开发自测、测试
5. 上线前进行安全检测

## 五、合规要求

### 话术合规

| 要求 | 说明 |
|---|---|
| 拒绝类话术 | 不透露具体拒绝原因，避免合规风险 |
| 额度类话术 | 不承诺具体额度数字（除非已确认） |
| 时效类话术 | 时间表述需准确，不虚假承诺 |
| 引导类话术 | 不强制引导，需尊重用户意愿 |

### 数据合规

| 要求 | 说明 |
|---|---|
| 数据来源合规 | 使用的三方数据产品需有合规授权 |
| 数据存储合规 | 数据存储周期和方式需符合法规 |
| 数据传输合规 | 数据传输需加密，防止中间人攻击 |
| 数据销毁合规 | 过期数据需按规范销毁 |

## 六、常见安全风险与防护

| 风险类型 | 防护措施 |
|---|---|
| 信息泄露 | 脱敏 + 权限隔离 + 加密传输 |
| XSS攻击 | 输入校验 + 输出编码 |
| SQL注入 | 参数化查询 + 输入校验 |
| 未授权访问 | 权限校验 + Token验证 |
| 越权操作 | 数据隔离 + 操作权限矩阵 |
| 大模型越界输出 | 话术审核 + 内容过滤 + 人工兜底 |
| 规则误杀 | 规则审批 + 效果追踪 + 回滚机制 |
