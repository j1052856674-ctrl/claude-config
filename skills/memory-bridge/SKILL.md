# Memory Bridge Skill

跨项目记忆同步与检索工具。它从各项目本地 `memory-hub/` 拉取长期记忆，结构化沉淀到长期知识库 `E:\个人仓库\03_Knowledge\记忆中枢`，并更新长期知识库索引。配置仓库 `E:\claude-config-master\memory-hub` 仅保存 UAM 协议、迁移状态和配置系统决策，不作为所有项目事实的长期倾倒区。

## 触发条件

- 用户输入 `/memory-bridge` 或“同步记忆”。
- 用户提到“沉淀记忆”“同步项目记忆”“更新全局索引”。
- 用户输入“检索记忆”“搜索跨项目经验”“查踩坑记录”“有没有类似 bug” → 进入经验检索模式。

## UAM 权威边界

- **项目事实权威源**：各项目自己的 `<project>/memory-hub/`。
- **长期可复用知识权威源**：`E:\个人仓库\03_Knowledge\记忆中枢\`。
- **运行时适配层**：`~/.claude/`、`~/.codex/`，只作为缓存、适配器或历史导入源。
- **配置仓库 memory-hub**：`E:\claude-config-master\memory-hub\`，只保存协议、迁移和配置系统决策。

> 若发现 legacy `.claude/memory/` 记录，可作为导入来源读取，但沉淀结果必须回写到项目本地 `memory-hub/` 或长期知识库，而不是继续扩大 legacy 目录。

## 前置配置

优先读取配置仓库中的同步清单：

```text
E:\claude-config-master\memory-hub\manifests\sync.example.json
```

也允许使用机器本地配置文件（不提交 Git）：

```text
memory-hub-sync.json
```

建议配置结构：

```json
{
  "vault_path": "E:/个人仓库/03_Knowledge/记忆中枢",
  "projects": {
    "E:/some-project": "项目显示名"
  }
}
```

若配置缺失，允许 fallback：

1. `vault_path` 缺失 → 从当前工作目录向上或从常用知识库路径探测 `03_Knowledge/记忆中枢/`。
2. `projects` 缺失 → 扫描已知项目根目录或 legacy Claude project cache，通过 git remote、目录名、项目 `memory-hub/MEMORY.md` 推断项目名。
3. 自动探测结果必须在报告中提示用户固化到本地配置文件；不得静默写入配置仓库。

## 同步沉淀流程

### 1. 确认项目列表并锁定路径

读取配置后，所有文件操作必须使用明确的绝对路径；禁止依赖当前工作目录，因为用户可能在任意项目中触发 `/memory-bridge`。

### 2. 扫描项目本地记忆

对每个项目递归扫描：

```text
<project>/memory-hub/**/*.md
```

包含常见子目录：

- `lessons/`
- `decisions/`
- `status/`
- `reviews/`
- `refs/`
- `arch/`
- `_archive/`

处理规则：

- 自动跳过 `_archive/` 中的文件，除非用户明确要求历史回溯。
- `bridge: false` → 跳过。
- `bridge: true` → 纳入候选。
- `bridge` 字段缺失 → 视为 true，遵循 UAM bridge 默认值规则。
- 首次迁移场景可全量扫描，但仍必须展示候选并等待用户确认后写入长期知识库。

### 3. 分类路由与价值排序

按内容特征路由到长期知识库中的对应主题区：

| 记忆特征 | 推荐目标 | 目标类型 |
|---|---|---|
| 踩坑、bug、错误、编译、环境、参数、兼容、崩溃、失败 | 通用经验 / 项目记忆 | 方法 |
| 偏好、反馈、不喜欢、喜欢、习惯、风格 | 反馈偏好 | 概念 |
| 项目状态、进度、架构、设计、计划、里程碑 | 项目记忆 | 项目 |
| Skill、模板、评审、规范、Agent 协作 | Skill设计 | 方法 |

多规则匹配优先级：`Skill设计 > 反馈偏好 > 通用经验 > 项目记忆`。

“通用经验”与“项目记忆”的判断标准：

- `source_projects` 数组长度 ≥ 2 → 通用经验。
- 内容中出现 2 个以上不同项目名 → 通用经验。
- 能提炼为跨项目方法论 → 通用经验。
- 只对单一项目阶段状态有效 → 项目记忆。

价值等级：

- **强推**：跨项目通用经验，与当前工作方式或工具链强相关。
- **关注**：项目特定，但有明确可迁移方法论。
- **暂缓**：纯项目状态，短期有用但长期复用价值低。

### 4. 展示候选并等待确认

写入长期知识库前必须展示候选列表，不自动全量写入：

```markdown
| # | 来源项目 | 记忆标题 | 价值等级 | 推荐目标 | 与已有笔记关系 |
|---|---|---|---|---|---|
| 1 | AI Worker Hub | Claude Code 参数不继承 | 强推 | 通用经验 | 合并到 Claude-Code工具踩坑.md |
| 2 | PRD 项目 | Mermaid 渲染异常 | 关注 | 项目记忆 | 新建 |
```

用户确认后才执行写入。若用户要求“全自动”，也必须先说明低价值内容污染长期知识库的风险，并给出可回滚路径。

### 5. 去敏化与通用化

写入前必须执行去敏检查：

- [ ] 移除或泛化项目特定绝对路径。
- [ ] 移除内部业务术语，替换为通用描述。
- [ ] 移除敏感配置，包括 API Key、密码、Token、私钥、PII。
- [ ] 保留逻辑骨架、触发条件、根因、修正方式和防范规则。

通用化示例：

```text
某项目 Cookie 封禁 → 30 秒间隔有效
```

提炼为：

```text
API 限流应对：当服务存在封禁或节流策略时，优先记录触发条件、最小安全间隔和退避策略。
```

### 6. Dedup 与合并策略

写入前搜索长期知识库已有笔记：

- 文件名或标题近义。
- tags / synopsis 重叠。
- 正文关键段落匹配。
- 同一主题但不同来源项目。

合并规则：

1. 同主题优先合并，不碎片化新建。
2. 在已有笔记末尾追加来源小节，不替换旧内容。
3. 去重追加 `source_projects` 和 `source_memory`。
4. 更新 `updated` / `bridge_sync` 日期。
5. 若已有笔记接近长度上限，提示用户拆分，不强行膨胀。
6. 若新记录取代旧记录，在新记录 `supersedes` 中写入旧记录 id 或文件名。

### 7. 结构化写入

长期知识库笔记使用标准 frontmatter，并保留来源可追溯性：

```yaml
---
title: ""
type: 概念/方法/项目/主题地图
status: 生长中
quality: 3
tags: []
source_projects: []
source_memory: []
bridge_sync: "YYYY-MM-DD"
created: "YYYY-MM-DD"
updated: "YYYY-MM-DD"
synopsis: ""
related: []
---
```

内容要求：

- 不复制大段项目原文，只保留可复用结论和必要证据指针。
- 添加双链到已有相关永久笔记。
- 记录来源项目和原始 `memory-hub/` 文件路径，便于追溯。

### 8. 更新索引

根据写入目标更新对应索引：

- 长期知识库 MOC：新增或更新沉淀笔记条目。
- 全局事实索引：必要时更新 `~/.claude/MEMORY.md` 的一句话摘要和详情指针。
- 项目本地索引：在源项目 `memory-hub/MEMORY.md` 中标注已沉淀去向，避免重复推荐。

索引只放短摘要和指针，不放正文。

### 9. 输出同步报告

报告必须包含：

- 本次扫描的项目数和记忆文件数。
- 用户选择写入、跳过、暂缓的候选。
- 新建了哪些笔记，合并了哪些笔记。
- 更新了哪些索引文件。
- 是否发现敏感信息、冲突或无法访问的路径。

## 经验检索模式

触发：用户输入“检索记忆”“搜索跨项目经验”“查踩坑记录”“有没有类似 bug”等查询意图。

### 检索流程

1. 解析关键词、分类范围和时间范围。
2. 在长期知识库中搜索文件名、正文、tags、synopsis。
3. 命中后读取 frontmatter，提取 `source_projects`、`source_memory`、`bridge_sync`。
4. 输出命中列表和关键发现摘要。
5. 用户要求查看来源时，再读取对应源项目 `memory-hub/` 文件。

输出示例：

```markdown
## 🔍 检索结果：「参数不继承」

| # | 笔记标题 | 分类 | 来源项目 | 最后同步 | 匹配度 |
|---|---|---|---|---|---|
| 1 | Claude Code 工具参数踩坑 | 通用经验 | AI Worker Hub, PRD 项目 | 2026-06-08 | 高 |

### 关键发现摘要
- ...

### 来源追溯
- 原始记忆文件：memory-hub/lessons/claude-code-param-loss.md
```

## 注意事项

- 可在任意项目中触发，但所有文件操作必须使用配置解析出的绝对路径。
- 去敏化是强制步骤；发现敏感信息时只能脱敏记录，不能输出完整敏感值。
- 自动探测到的项目列表应提示用户固化到本地配置文件。
- 配置仓库 `memory-hub/` 只保存协议和配置系统状态，不承载普通项目事实。
- 首次运行可以全量扫描，但不等于自动全量写入；长期知识库写入仍需用户确认。
