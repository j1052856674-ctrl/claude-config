# 骨架优先规范（4A）

> frontend v3 参考数据。SKILL.md Step 3 骨架生成时按需加载。

## 核心原则

骨架阶段只做布局结构 + 占位内容。**不写真实文案、不添加交互、不调视觉细节。** 目标：用户在 30 秒内确认"这个布局方向对不对"。

## 骨架产出标准

### 必须包含
- 完整 HTML 布局结构（所有 Section 的位置、容器、网格）
- `:root` CSS 变量块（Step 2.7 设计简报的 Token 已注入）
- 占位图片：灰色块 + 尺寸标注（如 `300×200`），使用 `bg-gray-200` 或 `var(--color-border)` 浅色
- 占位文案：lorem ipsum 或 `[Section名称文案待填充]`
- 真实标题占位符（用最终标题，方便判断信息层级）
- 中文字体栈

### 禁止包含
- 真实配图 / Unsplash 链接 / 占位图服务 URL
- 交互逻辑（onClick、表单提交、Dropdown 展开）
- 动效（transition、animation、GSAP）
- 精细配色（只用 CSS 变量默认值，不调色板微调）
- 真实数据（数字、价格、日期）

## 骨架验证门禁

必须全部 ✅ 才能进入 Step 4：

- [ ] 四断点不掉落（375 / 768 / 1024 / 1440）
- [ ] Section 顺序符合设计简报
- [ ] 所有 `var(--xxx)` 引用有效（无未定义 Token）
- [ ] 无 `h-screen`（已用 `dvh`）
- [ ] 固定元素有 `safe-area-inset`
- [ ] `<meta name="viewport">` 存在
- [ ] 中文字体栈存在
- [ ] 信息层级可辨识（标题 > 正文 > 辅助文字 的视觉差异明显）

## 确认流程

```
AI 生成骨架 → 写入 .claude/preview/index.html
  → 告知用户："🏗️ 骨架已生成 → http://localhost:3333/index.html，请确认布局方向"
  → 用户反馈：
    ✅ "可以" / "继续" → Step 4
    🔧 "XX 区域不对" → 修改骨架 → 重新写入 → 用户再次确认
    🔄 "换个布局思路" → 换一种 Section 排列方式 → 重新写入
```
