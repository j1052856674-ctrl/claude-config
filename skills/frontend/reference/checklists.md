# 交互模式速查 + 场景门控组件清单

> frontend v3 参考数据。SKILL.md Step 4 代码生成前按需加载。

## 交互模式速查

| Section | 适用场景 | 推荐交互模式 | 避免 |
|---------|---------|------------|------|
| **联系区** | 个人网站 / Portfolio | 展示地址文字 + 图标复制按钮（不跳转、不 mailto:） | mailto: 链接、大号"复制"按钮、行间分割线 |
| **联系区** | SaaS Landing / 企业站 | 联系表单 或 mailto: + 社交媒体图标链接 | 纯文字地址无交互 |
| **Footer** | 个人网站 / Portfolio | 极简：一行版权 + 社交图标（无多列链接） | 多列站点地图、重复导航 |
| **Footer** | SaaS / 企业站 | 4-5 列链接组 + Newsletter 订阅 + 版权行 | 单行极简（信息密度不足） |
| **CTA 区** | Landing 页末 | 一个大主按钮 + 一个次按钮 + 一句信任标签 | 3+ 按钮并排、无信任标签 |
| **导航** | 单页滚动站 | 无顶部导航（锚点滚动即导航），或 sticky 极简条 | 多级下拉菜单、汉堡菜单 |
| **导航** | 多页站 | sticky 顶栏 + 核心链接 + CTA 按钮 | 超过 6 个导航项、无 CTA |
| **项目/作品卡片** | Portfolio | 统一行模板：左侧元信息 + 右侧内容，同类型 Section 共享 | 不同 Section 各用一种卡片风格 |
| **技能/标签区** | 个人网站 | 分组标签云（同类标签视觉统一） | 无分组的平铺列表、进度条/百分比 |
| **社交链接** | 任何场景 | 图标 only（无文字标签），hover 品牌色 | 图标 + 大字标签、垂直排列 |

## 场景门控组件清单

### 通用必须（所有场景）

- [ ] 每个按钮有 `onClick` handler 或 `type="submit"`
- [ ] 每个 Modal/Dialog：X按钮 + Esc + 点击遮罩（三种全有）
- [ ] 每个破坏性操作：AlertDialog，不依赖浏览器 `confirm()`
- [ ] 每个表单：`onSubmit` + loading + error + success
- [ ] 每个 Dropdown/Menu：打开 + 关闭（点击外部）+ 键盘导航（↑↓ Enter Esc）
- [ ] 每个 Tabs：点击切换 + 键盘导航（←→）
- [ ] 响应式四断点不掉落：375px / 768px / 1024px / 1440px
- [ ] 图片 `alt` / icon按钮 `aria-label` / 输入框 `<label>` 关联
- [ ] 输入框不禁用粘贴（不设 `onPaste` 拦截）
- [ ] 空状态有明确下一步操作
- [ ] 可点击元素有 `cursor-pointer` + hover 反馈
- [ ] 触摸目标 ≥ 44px

### Dashboard 附加

- [ ] 侧边导航（可折叠）或顶部导航 + 主内容区
- [ ] 表格/卡片列表：loading 骨架屏 + empty + error 重试
- [ ] 搜索框：输入 → 触发搜索 → 展示结果（含空结果状态）
- [ ] 数据表格 `font-variant-numeric: tabular-nums`
- [ ] 高密度时去掉卡片容器，用分割线区分

### Landing 附加

- [ ] Hero 适配首屏：标题 ≤2行/≤8词，副标题 ≤20词，CTA 可见无需滚动
- [ ] Hero 顶部 padding ≤ pt-24，内最多 4 个文字元素
- [ ] Hero 内禁止：CTA下方小标签、信任条、功能列表、社交证明头像行
- [ ] CTA 按钮：≤3词/标签，桌面端不换行
- [ ] 无重复意图 CTA，CTA 链接有效目标
- [ ] 至少一个社交证明区（客户 Logo / 数据指标 / 用户证言）
- [ ] Logo墙在 Hero 下方独立区域，只用 logo

### Creative 附加

- [ ] 视觉个性明确（选定风格在整体布局中可辨识）
- [ ] 非对称布局（极简白除外）
- [ ] 允许实验性排版（大字号跨度、非标准网格）
- [ ] 至少一处视觉记忆点

### 展示/PPT 式附加

- [ ] 大标题 + 极少正文（每屏 ≤3 句话）
- [ ] 强对比视觉层级
- [ ] 每屏一个核心信息
