# 品类默认架构速查表

> frontend v3 参考数据。SKILL.md Step 2.0 Brief Inference 时按需加载。

## 速查表

| 品类 | 触发词 | 架构形态 | 数据能力 | 默认 Section 顺序 | 默认理由 |
|------|--------|:--:|:--:|------|------|
| 个人网站 | 个人网站、个人主页、about me | 单页 | 纯静态 | Hero → 关于我 → 项目/作品 → 经验/技能 → 联系 | 内容量 3-5 屏，单页滚动体验好；纯静态零费用零维护 |
| Portfolio | portfolio、作品集、showcase | 单页 | 纯静态 | Hero → 精选作品(grid) → 关于 → 联系 | 一页扫完比分类导航高效；GitHub Pages 免费托管 |
| SaaS Landing | SaaS、产品页、官网、landing | 单页 | 纯静态 | Hero → 功能 → 如何工作 → 定价 → 客户案例 → CTA → Footer | 说服漏斗从认知→兴趣→行动一气呵成 |
| 技术博客 | 博客、blog、技术文章 | 多页 | 需CMS/SSG | 首页(文章列表) → 文章详情 → 关于 → 归档 | 每篇文章独立长文，需独立页面 |
| 文档站 | 文档、docs、API文档 | 多页 | 纯静态 | 首页 → 快速开始 → 指南 → API参考 | 按主题分页业界标准；静态生成足够 |
| Dashboard | 后台、管理面板、dashboard | 单页(SPA) | 需后端API | 侧边栏 → 数据概览 → 详情表 → 设置 | SPA 避免数据页整页刷新 |
| 活动/会议页 | 活动、大会、meetup | 单页 | 纯静态 | Hero(日期+CTA) → 嘉宾 → 议程 → 赞助 → FAQ | 信息有限，单页保证不迷路 |
| 在线简历 | 简历、resume、CV | 单页 | 纯静态 | Header → 经历 → 技能 → 项目 → 教育 | 简历逻辑搬到网页，单页最自然 |
| 产品更新日志 | changelog、更新日志 | 单页 | 纯静态/需后端 | 时间线列表 → 订阅CTA | 排序列表，单页时间线最直观 |
| 状态页 | status、服务状态、uptime | 单页 | 需后端API | 总体状态 → 服务列表 → 历史事件 | 状态数据需实时查询 API |

## 使用规则

1. 命中品类 → 输出 🏗️ 架构判读（含默认理由摘要），展示给用户确认
2. 用户确认 → 进设计选配
3. 用户要求调整 → 以用户说的为准，调整后再次展示确认
4. 品类不在表中 → 最多问一个问题，拿到答案后同样展示确认
